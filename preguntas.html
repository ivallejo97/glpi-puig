Preguntas frecuentes

Conoce la respuestas a las preguntas mas comunes sobre GLPI

Generales

¿Que puedo hacer con GLPI?

Con GLPI puedes organizar y gestionar los activos TI y los servicios de soporte de ayuda de tu organización.

Puedes registrar todos los dispositivos de red (switches, routers, servidores, firewalls ...) y equipos finales (ordenadores portatiles, torres, impresoras...), y de esta manera conocer y tener controlada la infraestructura de red de tu organización.

Con GLPI puedes crear tickets de incidencias para reportar problemas o realizar peticiones.

También puedes llevar un control de las licencias de software de las que dispones y gestionar los contratos y proveedores de tu organización.

Y además también puedes integrar GLPI con otros sistemas, como sistemas de monitoreo de red, herramientas de inventario, sistemas de autenticación ...

Todas estas funciones se pueden llegar a automatizar.

En general con GLPI puedes gestionar toda la infraestructura de tu organización, reportar problemas, guardar documentación relevante para lo organización y tener controlado en todo momento lo que pasa dentro de tu organización.

¿Que tareas puedo automatizar con GLPI?

Con GLPI puedes automatizar varias funciones basicas como la asignación de tickets, generación de informes, envio de notificaciones, recordatorios de mantenimiento, actualizaciones de software, copias de seguridad, escaneos de seguridad ...

¿Como registro los dispositivos y equipos de mi organización en GLPI?

Para registrar los dispositivos finales, equipos de red y demas en GLPI, existe un agente GLPI que se debe activar en cada cliente o equipo de red , al activar este agente el dispositivo se registrara correctamente en el servidor GLPI y se podrán ver sus caracteristicas como nombre de equipo, dirección IP, MAC ...

¿Como activo el agente GLPI en los dispositivos y equipos de red de mi organización?

Para activar el Agente GLPI en un dispositivo puede variar según el tipo de sistema operativo que este usando este, en el caso de sistemas operativos Linux (ubuntu) para activar el agente GLPI se puede hacer mediante unos comandos, en el caso de sistemas operativos Windows lo puedes hacer con un programa grafico que se puede descargar en la pagina de GLPI, en ambos caso para que el dispositivo se sincronize correctamente con el servidor de GLPI se debe poner la IP del servidor durante la instalación del agente GLPI en el cliente o equipo de red.

En el caso de los equipos de red, para instalar el agente GLPI en la mayoria se debera hacer siempre por comando, ya que la mayoria de equipos de red solo tienen interfaz de comandos.




Instalación

¿En que sistemas operativos puedo instalar GLPI?

GLPI se puede instalar en varios sistemas operativos, en el caso de esta página nuestras instalaciones automaticas son para los sistemas operativos de Windows (10) y Linux (Ubuntu 22.04), aunque también se puede instalar en otros sistemas operativos como Red Hat, Mac Os, Solaris ...

¿Que necesito para instalar GLPI?

Para instalar GLPI necesita un servidor web como Apache, una base de datos que controle los datos de GLPI, los archivos de descarga de GLPI y PHP para que el servidor web pueda interpretar los archivos de GLPI.

Aunque en esta página puede encontrar todo lo necesario para instalar GLPI de manera automatica tan solo ejecutando tres comandos que se detallan en la guia de instalación.

Soporte

¿Como creo un ticket en GLPI?

Para crear un ticket en GLPI primero hay que iniciar sesión en GLPI con tu usuario, ir al apartado de soporte -> Crear Petición y por ultimo rellenar la petición y definir el nivel de urgencia de esta.

¿Que tipos de ticket puedo crear?

Existen dos tipos de ticket a crear:

Incidencia: Para crear tickets relacionados con problemas

Petición: Para crear tickets relacionados con peticiones de recursos.

¿A quien se le asigna el ticket?

Dependiendo del tipo de usuario que crea el ticket se le asigna a un técnico o a otro, por ejemplo un usuario que tenga todos los permisos podrá decidir a quién se le va a asignar el ticket

También se pueden crear reglas de asignación de tickets para que estos se asignen según la carga de trabajo en el servidor, es decir si un usuario tiene asignado 5 tickets y otro 3 el ticket se asignara automaticamente al técnico con 3 tickets asignados.




GLPI Cloud

¿GLPI ofrece servicios en la nube?

Sí, GLPI ofrece una opción de gestión en la nube, para mas información consulta esta página: https://glpi-network.cloud/

¿Que ventajas proporciona tener GLPI en la nube?

Las principales ventajas que se obtienen al implementar GLPI en la nube son el acceso remoto desde cualquier lugar, una mayor capacidad escalabilidad para ajustarse a las necesidades cambiantes, simplifica el mantenimiento al dejar la responsabilidad de la infraestructura en manos del proveedor.

Además, proporciona respaldo y recuperación de datos, costos predecibles, actualizaciones automáticas y liberación de recursos internos para tareas más estratégicas, lo que en conjunto mejora la eficiencia operativa y la seguridad de la plataforma.




Asistencia Profesional

¿Ofrece GLPI asistencia profesional?

Si, GLPI ofrece varios planes de soporte profesional que varian según el precio y las necesidades actuales de la organización que quiera obtener soporte sobre GLPI, para más información puedes consultar la siguiente página web: https://glpi-project.org/es/precios/
